# 🤟 SignBridge — Real-Time Sign Language Recognition

> A real-time American Sign Language (ASL) finger-spelling recognition system built with MediaPipe, OpenCV, and a Random Forest classifier. Detects hand landmarks via webcam, predicts signed letters with confidence scoring, and converts them to speech.

---

SignBridge/

## 🚀 How to Run (Step-by-Step)

Follow these three steps **in order**.

---

### Step 1 — Collect Training Data

Run the data collector to build your hand landmark dataset.

```bash
python src/Data_Collection.py
```

**How to use:**
- A webcam window will open with live hand tracking.
- Make an ASL letter sign with your hand.
- Press the **corresponding letter key** on your keyboard to record that frame.
- Repeat for each letter you want to train (aim for **100–200 samples per letter** for best accuracy).
- Press **`ESC`** to stop and save.

This generates `data/dataset1.csv`.

> ⚠️ Make sure to move/rename the output file to `data/dataset1.csv` before proceeding, or update the path in `Model_Train.py`.

---

### Step 2 — Train the Model

Train the Random Forest classifier on your collected dataset.

```bash
python src/Model_Train.py
```

**What it does:**
- Loads `data/dataset1.csv`
- Splits data 80/20 into training and test sets
- Trains a **100-tree Random Forest** classifier
- Prints the accuracy on the held-out test set
- Saves the trained model to `models/sign_lang_model.pkl`

**Expected output:**
```
1. Loading dataset...
Loaded XXXX total frames of hand signs.

2. Training the Random Forest Classifier...
3. Evaluating model on hidden test data...
------------------------------
Model Accuracy: XX.XX%
------------------------------

4. Exporting trained model to sign_lang_model.pkl...
SUCCESS! Your model is ready for deployment.
```

---

### Step 3 — Run Real-Time Detection

Launch the live sign recognition interface.

```bash
python src/main.py
```

**How it works:**
| Control | Action |
|---|---|
| **Show a sign** | Hold it steady for **1.5 seconds** to auto-add the letter |
| **`SPACE`** | Manually add the currently detected letter |
| **`ENTER`** | Speak the buffered word aloud via TTS and add it to the sentence |
| **`BACKSPACE`** | Delete the last letter in the buffer |
| **`C`** | Clear the entire buffer and sentence |
| **`Q`** | Quit the application |

**UI Explained:**
- **Large letter** — The sign currently being detected
- **ACC % bar** — Confidence score; turns green when above the 75% threshold
- **Locking timer** — Progress bar showing how long you've held the sign
- **BUFFER** — The word being built, letter by letter
- **OUT** — The full sentence output so far

## 🧠 How It Works

```
Webcam Frame
     │
     ▼
MediaPipe Hands  ──►  21 Landmarks (x, y, z each)
     │
     ▼
Normalize to Wrist  ──►  63-value feature vector
     │
     ▼
Random Forest Model  ──►  Letter Prediction + Confidence
     │
     ▼
Dwell Logic (1.5s hold)  ──►  Auto-add to word buffer
     │
     ▼
espeak TTS  ──►  Spoken word output
```

1. **MediaPipe** detects 21 hand landmarks per frame (x, y, z coordinates).
2. All coordinates are **normalized relative to the wrist** (landmark 0) to make predictions position-invariant.
3. The resulting **63-dimensional vector** is fed to the trained Random Forest.
4. A prediction is only accepted if its confidence exceeds the **75% threshold**.
5. A letter is added to the word buffer only after it is **held steadily for 1.5 seconds** (dwell input), preventing accidental letters.
6. The completed word is spoken aloud using **espeak**.

## 🔧 Configuration

Key parameters can be tuned at the top of `main.py`:

| Variable | Default | Description |
|---|---|---|
| `CONF_THRESHOLD` | `0.75` | Minimum confidence (0–1) required to accept a prediction |
| `SECONDS_TO_HOLD` | `1.5` | Seconds a sign must be held to auto-add the letter |
| `max_num_hands` | `1` | Number of hands tracked simultaneously |
| `min_detection_confidence` | `0.7` | MediaPipe hand detection sensitivity |

## 📊 Tips for Better Accuracy

- Collect at least **100 samples per letter** — more is better.
- Record in **varied lighting** conditions to improve robustness.
- Keep your hand **fully visible** in the frame during data collection.
- Use a **plain background** when collecting data.
- If accuracy is low for specific letters, **delete and re-collect** those classes.

## 🛠️ Troubleshooting

| Problem | Fix |
|---|---|
| `sign_lang_model.pkl not found` | Run `Model_Train.py` first to generate the model |
| `dataset.csv not found` | Run `Data_Collection.py` first and verify the output path |
| Camera not opening | Check that your webcam is connected; try changing `VideoCapture(0)` to `VideoCapture(1)` |
| `espeak` not working | Run `sudo apt-get install espeak` |
| MediaPipe install error | Try `pip install mediapipe==0.10.9` for a stable version |
| Low accuracy | Collect more samples per letter, especially for confusable signs |
